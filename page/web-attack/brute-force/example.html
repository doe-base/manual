Brute-force attacks are used in penetration testing to test the strength of authentication mechanisms. This guide is for **educational and legal** purposes only.

Step 1: Identify a Login or Input Field

Look for login pages, search fields, or hidden form inputs. Use tools like Burp Suite or browser DevTools to inspect where and how data is sent.

Step 2: Intercept the Request

Use Burp Suite (Proxy tab) to capture the login POST request. Send it to Intruder or Repeater for testing.

Step 3: Choose or Create a Wordlist

Use common password lists like rockyou.txt or generate custom lists using CeWL or Crunch.

Step 4: Launch the Attack

  • With Burp Intruder: Set positions, load payloads, and run the attack.
  • With Hydra (CLI):
    hydra -l admin -P rockyou.txt target.com http-post-form "/login:username=^USER^&password=^PASS^:Invalid"

Step 5: Analyze the Response

Look for differences in status code, response length, or success messages to spot valid credentials.
