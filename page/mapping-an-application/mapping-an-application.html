Mapping An Application

Step 1 – Scraping the Application

Gather as much information about the application as possible. This includes:

  • Application Pages vs Functional Paths
  • Every Link and Form
  • robots.txt and other exposed resources

Tools you can use:

  • 🔍 Web spidering tools – Still searching for a free one. Most good ones like Burp Spider are paid.
  • 🕹️ User-Directed Spidering – Currently using webhttrack for manual crawling
  • 🛠️ IEWatch and IE Dev Mode – Handy for legacy applications

Step 2 – Discovering Hidden Content (Go deeper)

  • 🔐 Brute-force Techniques – (Paid Burp Spider)
  • 🔍 Inference from Published Content
  • 🌐 Use of Public Info – Like GitHub repo leaks
  • 🖥️ Leveraging Web Server – Wikto
  • 🧩 Functional Paths – Burp Intruder
  • 🎯 Discovering Hidden Parameters

Step 3 – Analyzing the Application

Understand how the application works:

  • 🧪 Identifying Entry Points for User Input
  • ⚙️ Identifying Server-Side Technologies
  • 🛠️ Identifying Server-Side Functionalities
  • 📌 Mapping the Attack Surface
